AI 演进全解：
从「逻辑规则」到「生成创造」

AI 的工作方式和你写的任何业务代码都不一样。它不是被「写出来」的， 它是被「训练出来」的。但训练到底是什么意思？为什么早期的 AI 只会下棋， 而现在的 AI 能写代码、画画、做推理？

01
1950s – 1970s

第一阶段 逻辑推理阶段

符号主义黄金时代 · 图灵测试 · Minimax · ELIZA
核心信仰：智能 = 逻辑规则 + 路径搜索

这一代人怎么理解「智能」

1950 年，Alan Turing 发表了一篇论文，问了一个问题： 「机器能思考吗？」 当时的科学家给出了一个简洁的答案：人类之所以聪明，是因为我们会做逻辑推理。 所以，只要让机器也能做逻辑推理，机器就会变得聪明。

他们的核心思路是：把世界上所有的规则都写进去，然后让机器沿着规则推导答案。 就像一棵巨大的「决策树」——每个节点是一个判断，走到叶子节点，答案就出来了。

代表作一：ELIZA——「会聊天」的假心理医生

1966 年，MIT 的 Joseph Weizenbaum 做了一个程序叫 ELIZA。 很多第一次和它对话的人，真的以为自己在和一个理解自己感受的心理医生聊天。

但 ELIZA 根本不理解任何东西。它的工作原理分三步：

1
扫描关键词
程序扫描你输入的句子，找预设的关键词。比如发现了「母亲」这个词。
2
拆解句子结构
把句子套入模板：「我感到 [X]」→ 提取出 X = 「焦虑」。
3
填入回复模板
把提取的词塞进回复模板：「你为什么感到焦虑？能多说说吗？」输出给用户。
💡 程序员类比

ELIZA 本质上是一堆高级正则表达式 + 字符串模板替换。

用现代代码描述就是：识别 /我感到(.+)/，提取捕获组，拼接成 "你为什么感到" + 捕获组 + "？" 输出。

它创造了「智能的幻觉」——本质是极简的字符串操作，连数学都没用到。

代表作二：下棋 AI——「暴力搜索」决策

科学家让机器下跳棋、国际象棋。思路是：把所有未来可能的走法全部列出来， 形成一棵决策树，然后用算法找最优解。 核心算法叫 Minimax。

Minimax 算法：用一句话解释

「在所有我能走的选项里，选那个即便对手使出浑身解数来针对我，我依然能拿到最高分的走法。」

AI 有 A、B、C 三种走法：

走 A → 对手最优反击 → 你得 −100 分（死得很惨）

走 B → 对手最优反击 → 你得 −10 分（小亏）

走 C → 对手最优反击 → 你得 −5 分（最稳）

AI 选 C。通过递归遍历所有可能性，层层往回推，选最稳的路径。

但棋局的可能性是天文数字（国际象棋的变化数量超过宇宙中的原子数），不可能全算完。 所以工程师引入了启发式评估函数：算到一定深度后，用一个打分公式给当前棋盘状态打分， 而不是继续往下搜索。 

// 启发式评估函数（简化版国际象棋）
function evaluate(board) {
  let score = 0;
  score += myPieces.length * 10;        // 己方棋子数量×权重
  score += controlCenter(board) * 5;  // 控制中心区域加分
  score -= opponentPieces.length * 10; // 对方棋子数量扣分
  return score;
  // 不等算出所有结果，用「经验公式」近似最优解
}
⚡ 为什么这条路走不通
  • 规则集合不能穷举现实世界。现实有太多模糊、例外、上下文依赖。
  • 遇到规则之外的情况，系统直接崩溃或给出荒谬答案。问 ELIZA「今天天气怎么样」，它会把「天气」当关键词乱答一气。
  • 写规则的工程量是无限的。你永远写不完「关于这个世界的所有规则」。
↓

第一次撞墙：「通用逻辑」走不通。 试图用一套规则解决所有问题——这太狂妄了。 于是科学家们换了一个方向：「既然搞不出全能的神，那就造一个垂直领域的专业老师傅。」

02
1980s – 1990s

第二阶段 专家系统阶段

知识工程时代 · MYCIN · 知识库 · 推理机 · 第一次 AI 寒冬
核心信仰：智能 = 大量行业专业知识

这一代人怎么理解「智能」

1970 年代末，科学家们承认了失败：通用逻辑解决不了现实问题。 但他们注意到，人类在自己熟悉的领域里表现得非常出色—— 一个经验丰富的医生不是因为逻辑推理能力超强，而是因为他积累了大量医学知识。

新思路：我们无法让 AI 变成全知全能的神，但能不能让 AI 成为某个领域的「老专家」？ 把一个领域里所有专家的知识全部整理成规则，塞进系统，这不就是「垂直领域的专家」了吗？

架构：知识库 + 推理机

每个专家系统分成两个独立模块：

模块作用类比
知识库 由人类专家手动编写的成千上万条 If-Then 规则。比如：「如果患者发烧超过 39°C 且白细胞升高 → 怀疑细菌感染」 一本写满专家经验的大字典
推理机 接收用户输入，在知识库里逐条匹配规则，推导出结论 一个遍历字典、执行 if-else 的程序
💡 程序员类比

专家系统对后端开发者来说非常好理解：它就是一个超大型的业务规则引擎。

知识库 = 数据库里存的一堆 IF condition THEN action 规则记录。

推理机 = 一段不停 SELECT 规则、检查是否匹配、执行动作的代码。

整个系统的「智能」完全来自人类手动写进去的规则，AI 本身没有任何学习能力。

代表作：MYCIN 医疗诊断（1972 年）

MYCIN 包含超过 500 条医疗规则，用于诊断血液感染和推荐抗生素。 测试结果令人震惊：MYCIN 的诊断准确率达到 69%，超过了部分初级医生（65%）， 接近专科医生水平（80%）。

这是 AI 第一次在垂直领域真正有用。专家系统随后在医疗、地质勘探、金融风控等领域 大规模商业化，成为了第一波 AI 商业化浪潮。

⚡ 三大死穴，导致第一次 AI 寒冬
  • 维护地狱：规则从 500 条增加到 5000 条时，规则之间的相互冲突和冗余 让系统变成无法维护的「代码屎山」。修一条规则可能导致另外 50 条出问题。
  • 没有常识：遇到规则边界之外的情况，系统给出荒谬答案。 问医疗专家系统「患者因车祸骨折了」，它可能给你开个治血液感染的抗生素。
  • 知识获取瓶颈：请专家访谈、手动整理规则、再手动编写， 这个过程极其昂贵且缓慢。一个领域的知识库可能要花 5-10 年才能建完。
阶段小结

专家系统证明了「知识」在 AI 中的核心地位，但它暴露了一个残酷现实：

靠人类手动编写规则来模拟智能，效率和复杂度都是不可持续的。

AI 需要一种能够「自己学习」的方法，而不是依赖人类把世界上所有的知识一条条塞进去。

↓

第二次撞墙：「人工的智能」太贵太脆弱。 靠人工编写规则根本跟不上现实世界的复杂度。 新一代科学家提出了一个根本性的问题： 「能不能不告诉机器规则，而是给它看大量例子，让它自己总结规律？」

03
1990s – 2017

第三阶段 数据驱动阶段

连接主义崛起 · SVM · 深度学习 · AlexNet · AlphaGo
核心信仰：智能 = 算法 × 数据 × 算力

思维方式的根本转变

旧方式（专家系统）：工程师定义「什么是猫」：有四条腿、有毛、两只尖耳朵…… 然后写代码检查这些特征。

新方式（数据驱动）：直接给模型看 100 万张猫的图片，让它自己学「什么是猫」。 工程师不再定义规则，而是定义一个能从数据中学习的数学模型。

f(输入数据) ≈ 正确答案
找到一个函数 f，使得对所有训练样本，预测结果尽量接近正确答案。
这个「找函数」的过程，就是「训练」。

神经网络是怎么「学习」的

这是整篇文章最重要的内容之一。神经网络的学习过程如下：

1
前向传播（Forward Pass）——做一次预测
把一张狗的图片输入神经网络。网络内部有很多「神经元」，每层神经元对上一层的输出做加权求和， 再经过一个激活函数。最终输出一个预测：「猫 70%，狗 30%」。
2
计算损失（Loss）——量化错了多少
标准答案是「狗」，但模型预测狗只有 30%。「损失函数」计算出一个数字，代表这次预测有多错。 损失越大，模型越差。
3
反向传播（Backpropagation）——找谁的责任
算法从输出层往输入层「倒推」：这次预测错了，每个神经元的权重（参数）应该往哪个方向调整？ 调整多少？这个过程叫「梯度下降」。
4
重复百万次——模型越来越准
对训练集里的每一张图片重复这三个步骤，每次根据梯度微调参数。 经过几百万次迭代，模型的预测越来越准确。这就是「训练」。
💡 程序员类比：反向传播 = 自动 Debug

反向传播就像代码的自动化测试 + 自动修 Bug：

① 运行代码（前向传播）→ 测试不通过（计算损失）

→ 用 profiler 定位哪个函数负主要责任（反向传播计算梯度）

→ 自动微调那个函数的参数（梯度下降）→ 重跑测试

区别是：代码里你自己改 Bug；神经网络里，梯度下降算法自动帮你改参数，而且一次改几亿个参数。

两个改变历史的时刻

2012 年，AlexNet。 Geoffrey Hinton 团队在 ImageNet 图像识别大赛中，用深度卷积神经网络取得了 15.3% 的错误率，而第二名的传统算法是 26.2%。 这是断崖式的碾压。从这一刻起，深度学习称霸 AI 领域，再无争议。

2016 年，AlphaGo。 AlphaGo 击败了围棋世界冠军李世石（4:1）。 它不只是学习了人类的 16 万盘棋谱——它还通过强化学习自己和自己下了 3000 万盘棋， 发展出了人类从未想到过的走法。 这证明：机器通过数据积累出来的「直觉」，可以超越人类用逻辑推导出来的策略。

为什么这次成功了：三剑客同时到位

要素发生了什么影响
📦 大数据 互联网爆发，人类上传了数十亿张图片、文本，提供了前所未有的训练素材 模型终于有足够的「例子」可以学习
⚡ GPU 算力 NVIDIA 的显卡把原本需要几年算完的神经网络，压缩到几天内完成 训练成本从「不可能」变成「可以负担」
🧮 算法成熟 深度学习解决了多层神经网络难以训练的「梯度消失」问题（ReLU、BatchNorm 等） 网络可以做得越来越深，能力越来越强
⚡ 仍然没解决的问题
  • 每个模型只能做一件事：识别图片的模型不会翻译文字，翻译的模型不会下棋。训练一个新任务要从头来过。
  • 需要大量标注数据：每张图片都要人工标注「这是猫」「这是狗」，成本极高。
  • 只会「判断」，不会「创造」：这封邮件是不是垃圾邮件？这张图里有没有猫？它只能回答是/否，不能生成新内容。
↓

第三次转变：不是「撞墙」，而是「突破天花板」。 2017 年，Google 的一篇论文提出了 Transformer 架构， 彻底终结了视觉、语音、文本各用一套算法的乱象， 开启了「一个模型搞定一切」的新时代。

04
2017 – 至今

第四阶段 大模型与生成式 AI

Transformer · 预训练 · GPT · Claude · 多模态 · Agent
核心信仰：智能 = 巨量参数 + 海量数据 + 统一架构

Transformer：终结乱象的「大一统理论」

在 Transformer 之前，AI 的各个领域各自为政： 图像有图像的模型（CNN），文本有文本的模型（RNN/LSTM），语音有语音的模型， 互不兼容，互不通用。

2017 年，Google Brain 团队发表了一篇题为 《Attention Is All You Need》的论文。 Transformer 的核心洞见可以用一句话概括：

万物皆可向量化，万物皆可 Transformer。

文字、图片、声音、视频——只要能转换成数字向量（一串数字）， 就可以用同一个架构处理。这就是为什么现在的多模态模型能同时看图、听话、写文章—— 不是有什么魔法，而是因为所有数据都被统一成了同一种格式。

Attention 机制：AI 学会了「看重点」

Transformer 最核心的技术是「自注意力机制（Self-Attention）」。 先看一个让它变得有意义的例子：

一个例子理解 Attention

句子：「银行坐落在河边，它的利率很高。」

当 AI 处理「它」这个词时，它需要判断「它」指的是「银行（金融机构）」还是「河边的堤岸」。

传统模型顺序读取，读到「利率」时已经离「银行」太远，容易搞混。

Attention 的解决方案：在处理「它」这个词时，让模型同时去「关注」整个句子里的所有其他词，给每个词分配一个权重。

「它」和「利率」的关联权重高 → 这里的「银行」是金融机构。

「它」和「河边」的关联权重低 → 不是堤岸。

💡 程序员类比：Attention = 动态上下文查询

你可以把 Attention 理解为一个对整个序列做的动态 SQL 查询：

SELECT value FROM context WHERE relevance_to(current_word) > threshold ORDER BY relevance DESC

每次处理一个词，都对整个上下文做一次这样的「相关性检索」，获取最相关的信息加权融合。这让模型能处理超长距离的依赖关系，而 RNN 在这方面非常吃力。

预训练：AI 是怎么「读完整个互联网」的

大模型的训练分两个阶段，这是理解「ChatGPT 为什么什么都懂」的关键：

1
预训练（Pre-training）——「完形填空」学遍互联网
把整个互联网的文本（Wikipedia、GitHub、论文、书籍……）喂给模型， 让它做一件事：预测下一个词。 比如「今天天气___」→ 预测「很好」。

看起来很简单，但为了准确预测下一个词，模型被迫学会了：语法规则、历史事实、 逻辑推理、代码语法、甚至幽默感。 它不是被告知这些知识的，而是在「完形填空」的过程中自己总结出来的。
2
微调（Fine-tuning）——从「博学的野兽」变成「有礼貌的助手」
预训练完的模型什么都知道，但它只是在无脑续写文本——它可能续写出暴力、错误、有害的内容。

微调通过「人类反馈强化学习（RLHF）」改造它： 人工评估员对模型的多种回答打分，训练一个「奖励模型」学会人类偏好， 再用强化学习让大模型的回答越来越符合人类期望。 这一步把「博学的野兽」变成了「有用、无害、诚实的助手」。

生成式 AI 怎么「生成」内容——不是查数据库

这是最多人困惑的地方。模型不是在「查找」答案，而是在「逐词计算概率」然后「采样」。 

// 文本生成的伪代码（以 GPT 为例）
function generate(prompt) {
  let output = prompt;

  while (!isComplete(output)) {
    // 对词表里每个词计算概率
    const probs = model.predict(output);
    // probs 是一个长度为 10万+ 的概率分布
    // 例如: { "很": 0.40, "不": 0.20, "真": 0.15, ... }

    // 按概率采样（不是取最大值！）
    const nextToken = sampleFrom(probs, temperature=0.7);

    output += nextToken;
    // 重复，直到生成完整回复
  }

  return output;
}
💡 为什么每次问同样的问题，答案不一样？

因为生成过程包含随机性（概率采样），而不是确定性查找。

temperature 参数控制随机程度：

→ temperature = 0：每次都选概率最高的词，结果完全确定，但缺乏创意

→ temperature = 1：按概率分布随机采样，结果有创意但可能跑偏

→ temperature = 2：更随机，有时候会胡言乱语

这就是你在 API 调用时设置 temperature 参数的实际含义。

涌现（Emergence）：最诡异也最重要的现象

当模型规模从 10 亿参数增长到 1000 亿参数时，发生了一件奇怪的事： 模型突然表现出了从未被刻意训练过的能力—— 复杂的多步推理、类比推理、写高质量代码、理解幽默……

没有人明确知道为什么会这样。这就是「涌现」——能力不是被设计进去的，而是从规模中自发产生的。

💡 类比理解涌现

单个水分子没有「湿」的属性，但大量水分子聚在一起，就有了「湿」的性质。

单个神经元不会推理，但当 1750 亿个参数组合在一起，模型就「会」推理了。

规模本身创造了新的能力。这是 AI 领域目前最大的未解之谜之一。

开发者视角：你的工作范式变了

维度以前（数据驱动时代）现在（大模型时代）
调用方式传入结构化参数，得到结构化结果传入自然语言 Prompt，得到自然语言回复
核心工作写业务逻辑、调 SQL写 Prompt、设计 Agent 工作流
架构模式MVC / 微服务RAG（检索增强生成）/ Agent
处理模糊逻辑几乎不可能用代码实现直接用 AI 处理模糊判断
需要 AI 知识需要机器学习博士一个 API Key + 基本工程能力
RAG 架构：不训练模型，只「喂」私有知识

公司有大量私有文档（合同、手册、报告），但无法送给 GPT 训练（费用高昂且有隐私问题）。

RAG（检索增强生成）的流程：

① 把所有公司文档向量化，存入向量数据库（Pinecone、Weaviate）

② 用户提问时，先在向量数据库里检索最相关的几段文档

③ 把这几段文档 + 用户问题一起塞进 Prompt，发给大模型

④ 大模型根据这些上下文生成答案

你不需要训练模型，不需要 GPU，不需要机器学习知识。用你熟悉的后端技术就能构建出「能回答公司内部问题的 AI 助手」。

写在最后

AI 的每一次进化，都不是旧技术的「改进」，而是底层范式的彻底颠覆。 每一次颠覆，都是因为上一代撞了一面新的墙——规则写不完、专家经验难以规模化、 模型泛化能力差……

你真正需要的能力

作为开发者，你不需要成为 AI 科学家。

但理解这四个阶段的底层逻辑，能帮你在面对每一个新 AI 工具时，快速判断：

→ 它的能力边界在哪里？

→ 它的根本局限是什么？

→ 它适合解决什么类型的问题？

这是比「会用工具」更值钱的判断力。
